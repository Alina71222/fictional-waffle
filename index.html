Сегодня я расскажу, из каких частей состоит доменное имя моего сайта и на каких уровнях иерархии они находятся.

Мой сайт опубликован на GitHub Pages по адресу: https://alina71222.github.io/fictional-waffle/.

Однако само доменное имя — это только часть до первого слеша: alina71222.github.io.

Оно состоит из следующих уровней DNS-иерархии:

  • Корневой домен — неявная точка в конце («.»), технический уровень всей системы доменных имён.
  • Домен верхнего уровня (TLD) — .io. Изначально он был выделен для Британской индийской океанической территории, но сегодня широко используется в IT-проектах.
  • Домен второго уровня — github. Это основное имя сервиса GitHub.
  • Поддомен третьего уровня — alina71222. Это моё имя пользователя на GitHub, которое автоматически становится поддоменом в рамках GitHub Pages.

Таким образом, полное доменное имя: alina71222.github.io — это трёхуровневое имя, где мой аккаунт выступает в роли поддомена над службой github.io.

Часть /fictional-waffle/ — это не домен, а путь к проекту внутри репозитория на GitHub Pages.
